## 什么是VPS测速脚本，搬瓦工用户为什么需要它

VPS测速脚本就是一行命令，SSH登录进去粘贴回车，脚本自动帮你跑一遍服务器的基础性能测试，输出一份汇总报告。

搬瓦工VPS测速脚本的核心价值在于：机器拿到手，不靠官网标注的参数，自己实测一遍磁盘读写、三网延迟和带宽，才算真正摸清楚这台VPS的实际状态。特别是如果你在用CN2 GIA-E套餐，不同机房之间的体感差异相当大，测一遍就清楚了。

> **一句话定义**：VPS测速脚本是通过SSH一行命令触发的服务器综合性能测试工具，输出磁盘IO、CPU跑分、三网网速与延迟等核心指标，帮助用户直观评估VPS实际性能。

## bench.sh：最快上手的入门脚本

说实话这个脚本活了好多年了，一直在更新，很多老鸟的第一反应就是先跑它。

执行命令：

bash
wget -qO- bench.sh | bash
# 或者备用：
curl -Lso- bench.sh | bash


跑完会输出几块内容：

1. **系统基础信息**：CPU型号、内核版本、虚拟化架构（搬瓦工全线KVM），以及内存、硬盘大小
2. **磁盘IO速度**：三次测试取平均值，单位MB/s
3. **全球节点下载速度**：测你的VPS到各大数据中心的下载带宽

值得注意的是，bench.sh里的速度节点以国外为主，测的是VPS到世界各地的连接质量，而不是专门测到国内的速度。国内三网（电信/联通/移动）的回程要看别的脚本。

## YABS：认真对比性能时选这个

YABS相比bench.sh更偏向性能跑分，测试项更规范。核心测试三件套：

- **fio磁盘读写**：4K/64K/512K/1M不同块大小的随机/顺序读写，比bench.sh的IO测试精细很多
- **iperf3网络测速**：测到全球多个数据中心的上传下载速度
- **Geekbench**：CPU+内存性能的跑分，有单核和多核两个维度

执行命令：

bash
curl -sL yabs.sh | bash


如果只想快速看CPU跑分，不想等网络和磁盘测试（能省不少时间）：

bash
curl -sL yabs.sh | bash -s -- -if5


`-i` 跳过iperf3，`-f` 跳过fio磁盘，`-5` 指定跑GB5（Geekbench 5）。

在搬瓦工DC9套餐上跑YABS，磁盘读写速度比较稳定，iperf3到部分美国节点的带宽在旷达几百Mbps量级，CN2 GIA线路的延迟控制得住。不过要说明：不同宿主机节点之间确实有性能差异，同款套餐跑出来的数字可能不完全一样。

## 融合怪：想一次性看完所有指标就用这个

融合怪（ecs）是目前信息量最全的VPS测评脚本之一，对搬瓦工用户特别实用，因为它专门包含了**三网回程路由测试**——这是判断搬瓦工选哪个机房最关键的数据之一。

执行命令：

bash
bash <(wget -qO- --no-check-certificate https://gitlab.com/spiritysdx/za/-/raw/main/ecs.sh)


脚本会自动跑以下几项：

1. CPU/内存/磁盘基础信息
2. CPU性能测试（sysbench）
3. 内存读写速度
4. 磁盘fio读写
5. 流媒体解锁检测（Netflix、Disney+等）
6. **三网回程路由及延迟**（电信、联通、移动各三个节点）

### 怎么看三网回程结果

这一项是搬瓦工用户最需要看的。回程路由决定了你从国内访问这台VPS的实际体验：

- 看回程路由里有没有出现 **59.43.x.x / 59.45.x.x**（这是CN2 GIA的IP段），有的话说明电信走的是GIA线路
- 联通方向看有没有走 **AS9929** 或 **AS10099**（CUII/CUP精品网）
- 移动方向看有没有 **CMIN2** 或 **CMI**
- 延迟在中美线路上大概150-180ms是正常范围，晚高峰偶尔高一些；香港/日本机房延迟可以低到30-60ms量级

搬瓦工的一个特点是：买的CN2 GIA-E套餐支持在KiwiVM后台自由切换12个以上机房，所以可以先在默认机房跑一遍融合怪，看看三网回程，不满意直接切机房再测一次。

## 测速脚本结果怎么判断好不好

跑完脚本，大部分人盯着那堆数字不知道该怎么解读。说几个实用的判断标准：

**磁盘IO**：搬瓦工标注的是SSD + RAID-10，实测IO一般在300-600MB/s区间算正常，低于200MB/s可能是节点负载较重，可以考虑在KiwiVM迁一下机房再测。

**三网延迟**：CN2 GIA-E套餐的洛杉矶机房，对电信用户平均延迟在150ms上下，联通稍好，移动走CMIN2也不错。香港机房三网延迟普遍在50ms以内，差距确实明显。

**Geekbench跑分**：这个看单核分数就够了，搬瓦工常规套餐用的是Intel E5系列处理器，单核分数一般在400-700区间；新一代的AMD EPYC系列（部分限量版Box系列套餐在用）单核分数会高出不少。

**iperf3带宽**：CN2 GIA-E套餐标注带宽2.5Gbps，但这是上行最大值，实际测速和线路拥塞情况有关，晚高峰普遍会低一些，这是正常现象。

## 常见问题 FAQ

**Q：bench.sh 和 YABS 哪个更准确？**

A：各有侧重。bench.sh更轻量，主要看全球节点下载速度；YABS的fio磁盘测试更标准，iperf3网络测试更规范，想要严肃对比就用YABS。两个都跑也不费时间，就是前后几分钟的事。

**Q：融合怪脚本跑完时间比较长，能不能只测部分？**

A：可以。融合怪支持选择只跑某几项测试，执行脚本后会有交互菜单，按提示选你需要的项目就行。如果SSH连接不稳定，建议配合screen或tmux使用，跑完结果还会自动上传到pastebin生成分享链接。

**Q：搬瓦工VPS测速脚本跑出来慢，是不是机器有问题？**

A：不一定。测速的时间段影响很大，晚高峰（国内时间晚8-11点）是高峰期，延迟和带宽都会比凌晨测出来差一些。建议在几个不同时间段各跑一次，取平均情况判断。此外，如果三网回程测试显示线路走的不是GIA，可以在KiwiVM后台切换机房再测。

**Q：搬瓦工有没有自己的测速工具？**

A：有。搬瓦工官方有一个Looking Glass工具，可以直接在网页端测试到各个机房的ping延迟和路由路径，不需要SSH登录，适合购买前先对比一下各机房的网络质量。

**Q：测速脚本用完想换机房，怎么操作？**

A：登录KiwiVM控制面板（搬瓦工自研的VPS后台），在"Migrate to another DC"功能里选择目标机房，系统会自动完成迁移，IP地址会变，但数据保留。CN2 GIA-E套餐可选的机房超过12个，换完机房重新跑一遍测速脚本，对比一下再决定留哪个。
